À propos du projet

Groupie Tracker – Site web exploitant l'API IGDB

Présentation

Ce projet est un site web développé en Go qui exploite l'API IGDB (Internet Game Database) pour afficher les catalogues de jeux vidéo de différents studios : SEGA, Nintendo, Ubisoft et Level-5. Le site permet de parcourir, filtrer, rechercher et sauvegarder en favoris des jeux vidéo.

API et Endpoints utilisés

L'API utilisée est IGDB (api.igdb.com/v4), une API REST qui retourne des données au format JSON. L'accès nécessite un token d'authentification Twitch avec renouvellement automatique.

/v4/games

Récupérer les jeux par studio, les détails d'un jeu, et les DLCs associés.

/v4/search

Rechercher des jeux par nom via la barre de recherche.

OAuth Token

Authentification via l'API Twitch avec gestion automatique du renouvellement du token.

Fonctionnalités implémentées

FT1 – Recherche

Recherche sur le nom du jeu. Les résultats affichent la couverture, le nom et le résumé.

FT2 – Filtres

Filtrage cumulatif sur 3 propriétés : catégorie, genre et plateforme. Les filtres se combinent entre eux.

FT3 – Pagination

Affichage par lots de 20 jeux avec navigation entre les pages. Les filtres sont conservés lors du changement de page.

FT4 – Favoris

Ajout et suppression de jeux en favoris. Les données sont persistées dans un fichier JSON.

Bonus – Page 404

Page d'erreur thème rétro-gaming avec effets glitch, scanlines CRT et mini-jeu Dino.

Bonus – Token API

Gestion automatique du token IGDB avec renouvellement via l'API Twitch.

FAQ – Synthèse du projet

Le projet a été décomposé en plusieurs phases :

  • Phase 1 – Choix de l'API : J'ai choisi l'API IGDB car elle offre une grande richesse de données sur les jeux vidéo (genres, plateformes, couvertures, DLCs).J'ai testé les endpoints avec Postman pour valider les données retournées.
  • Phase 2 – Architecture : J'ai structuré le projet en suivant le pattern MVC (Models, Controllers, Services, Routers, Templates) pour garder un code organisé et maintenable.
  • Phase 3 – Pages de base : Création de la page d'accueil avec le carrousel tirer d'une video youtube, puis les pages de catalogue par studio (SEGA, Nintendo, Ubisoft, Level-5) avec l'affichage des jeux.
  • Phase 4 – Fonctionnalités : Implémentation des filtres, de la pagination, de la recherche et des favoris dans cet ordre.
  • Phase 5 – Finitions : Page 404 stylisée, mini-jeu Dino (petit bonus), page à-propos et documentation (README).

J'ai priorisé les fonctionnalités obligatoires avant les bonus :

  • Priorité haute : Connexion à l'API, affichage des jeux, filtres et pagination (FT1-FT3) car ce sont les fonctionnalités de base attendues.
  • Priorité moyenne : Favoris (FT4), recherche et page détails, qui enrichissent l'expérience utilisateur.
  • Priorité basse : Bonus visuels (page 404, mini-jeu Dino), documentation et finitions CSS.

Cette approche m'a permis d'avoir un site fonctionnel rapidement avant d'ajouter les améliorations.

Ma stratégie de documentation s'est basée sur plusieurs sources :

  • Documentation officielle IGDB : Pour comprendre les endpoints, les champs disponibles et le format des requêtes Apicalypse. J'ai jamais au tant lu.
  • Documentation Go : Pour les packages standards (net/http, html/template, encoding/json).
  • Postman : Pour tester et valider les requêtes API avant de les intégrer dans le code.
  • Cours Ynov : Les exemples du professeur ont servi de base pour implémenter les fonctionnalités.
  • Authentification API : La gestion du token Twitch avec expiration j'en pouvais plus j'ai trouver un indien qui ma aider avec une video
  • Filtres cumulatifs : Combiner plusieurs filtres (catégorie + genre + plateforme) tout en gardant les checkboxes cochées après soumission du formulaire.
  • Pagination + Filtres : Conserver les filtres sélectionnés lors du changement de page.
